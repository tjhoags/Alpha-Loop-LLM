# PRD - InstantMeet MVP

## Objective
Launch a payments-first scheduling tool with team routing and embeddable booking pages tonight.

## Users
- Organizer/Admin (creates meeting types, sets price, shares links)
- Attendee (books and pays for a slot)

## Core Requirements (Tonight)
1) Create meeting type with price and duration
2) Publish booking page
3) Accept booking (payment stub OK) and store in DB
4) Simple dashboard to create meeting type and view bookings

## Nice-to-Have (Later)
- Calendar sync (Google/Microsoft)
- Real Stripe Checkout and webhook handling
- Lead enrichment and CRM export
- Workspace roles and audit logs
