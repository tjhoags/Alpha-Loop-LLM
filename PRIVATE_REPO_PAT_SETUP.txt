STEP 1: MAKE THE REPOSITORY PRIVATE
================================================================================

1. Go to: https://github.com/tjhoags/alpha-loop-llm

2. Click "Settings" tab (top right of repo page)

3. Scroll down to "Danger Zone" section

4. Click "Change visibility"

5. Select "Change to private"

6. Type the repository name to confirm: alpha-loop-llm

7. Click "I understand, change repository visibility"

IMPORTANT: Once private, only you and collaborators can see the code.

================================================================================
STEP 3: CREATE A PERSONAL ACCESS TOKEN (PAT)
================================================================================

PATs are required for:
- Git operations (clone, push, pull) with HTTPS
- API access to GitHub
- Cursor IDE integration
- CI/CD pipelines

HOW TO CREATE A PAT:
--------------------

1. Go to: https://github.com/settings/tokens

2. Click "Generate new token" > "Generate new token (classic)"

3. Set token details:
   - Note: "Alpha Loop Capital - Cursor IDE"
   - Expiration: 90 days (or custom)
   
4. Select scopes (permissions):
   [X] repo (Full control of private repositories)
       [X] repo:status
       [X] repo_deployment
       [X] public_repo
       [X] repo:invite
       [X] security_events
   [X] workflow (Update GitHub Action workflows)
   [X] write:packages
   [X] read:org
   [X] gist

5. Click "Generate token"

6. COPY THE TOKEN IMMEDIATELY (you cannot see it again!)
   Example format: ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

7. Store the token securely:
   - Add to .env file
   - Save in password manager
   - DO NOT commit to git

================================================================================
STEP 4: CONFIGURE GIT TO USE PAT
================================================================================

OPTION A: Store credentials (Windows)
-------------------------------------
# Run this command
git config --global credential.helper manager

# Next time you push/pull, enter:
Username: tjhoags (or your GitHub username)
Password: [paste your PAT here]

Windows Credential Manager will save it.


OPTION B: Store credentials (Mac)
---------------------------------
# Use macOS Keychain
git config --global credential.helper osxkeychain

# Next push/pull will prompt for credentials
# Enter PAT as password


OPTION C: Add PAT to remote URL (per-repo)
------------------------------------------
# Update remote to include PAT (replace YOUR_PAT)
git remote set-url origin https://tjhoags:YOUR_PAT@github.com/tjhoags/alpha-loop-llm.git

Note: This stores PAT in .git/config (local to machine, not committed)


OPTION D: Environment variable
------------------------------
# Add to .env file
GITHUB_TOKEN=ghp_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# In Python:
import os
token = os.getenv("GITHUB_TOKEN")

================================================================================
PAT MANAGEMENT BEST PRACTICES
================================================================================

1. ROTATION: Create new PAT every 90 days
   - Set calendar reminder
   - Generate new token before old expires
   - Update all machines

2. SCOPE: Use minimum necessary permissions
   - For basic git: just "repo" scope
   - For CI/CD: add "workflow" scope

3. STORAGE: Never commit PAT to code
   - Store in .env (gitignored)
   - Use credential manager
   - Use password manager for backup

4. REVOCATION: If PAT is compromised
   - Go to https://github.com/settings/tokens
   - Click "Delete" on compromised token
   - Generate new token immediately
   - Update all machines

================================================================================
TROUBLESHOOTING
================================================================================

ERROR: "Repository not found"
-----------------------------
- Repo is private and you're not authenticated
- Enter PAT as password
- Or: Verify you're a collaborator

ERROR: "Authentication failed"
------------------------------
- PAT may be expired
- Generate new PAT
- Update credential manager

ERROR: "Permission denied"
--------------------------
- Check collaborator access level
- Need "Write" permission to push

ERROR: Cursor can't access repo
-------------------------------
- Sign out of GitHub in Cursor
- Sign in again
- Authorize private repo access
